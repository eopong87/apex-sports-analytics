# ApexEdge AI Sports Analytics Platform

Multi-region AWS sports analytics dashboard with live ESPN data, AI-powered insights, and full infrastructure-as-code deployment via Terraform.

## Live Demo
- Primary (us-east-1): http://apex-sports-analyst-2026-primary.s3-website-us-east-1.amazonaws.com
- DR Standby (us-west-2): http://apex-sports-analyst-2026-secondary.s3-website-us-west-2.amazonaws.com
- API: https://6ogkl5pzle.execute-api.us-east-1.amazonaws.com

## What This Project Demonstrates
- Multi-Region Disaster Recovery — Primary in us-east-1, warm standby in us-west-2
- Infrastructure as Code — 49 AWS resources provisioned with Terraform
- Serverless Backend — 5 Python 3.12 Lambda functions behind API Gateway
- Live Sports Data — Real scores from ESPN API (NBA, NFL, MLB)
- AI Analytics — Game analysis powered by Google Gemini
- Security — API keys in Secrets Manager, IAM least-privilege roles
- Cost Optimized — DynamoDB PAY_PER_REQUEST, EventBridge and Kinesis disabled by default

## Architecture

## AWS Services Used
- Amazon S3 — Static website hosting in two regions
- S3 Cross-Region Replication — Keeps secondary in sync
- AWS Lambda — 5 Python 3.12 serverless functions
- API Gateway — Routes GET requests to Lambda
- DynamoDB — 5 tables for games, stats, odds, insights
- Secrets Manager — Secure API key storage
- IAM — Least-privilege roles
- CloudWatch — Logs and monitoring
- EventBridge — Scheduled ingestion (disabled by default)
- Kinesis — Real-time streaming (disabled by default)
- Route 53 — Optional DNS failover

## Quick Start
# Clone the repo
git clone https://github.com/apex-sports-analyst/apex-sports-analytics.git
cd apex-sports-analytics

# Create your variables file
cp terraform/terraform.tfvars.example terraform/terraform.tfvars

# Edit bucket_prefix to something unique
# Then run:
cd terraform
terraform init
terraform plan
terraform apply

## API Routes

cd terraform

## Portfolio Summary
Built a production-architecture AI sports analytics platform on AWS using
a multi-region disaster recovery pattern. The frontend is hosted on Amazon S3
with cross-region replication from us-east-1 to us-west-2. A serverless backend
of five Python 3.12 Lambda functions serves live ESPN sports data through API
Gateway. AI-powered game analysis is generated by Google Gemini. All 49 AWS
resources are provisioned as infrastructure-as-code with Terraform, using
feature flags for cost-optimized deployments.
